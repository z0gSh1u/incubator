## 问题

在实现 AI 聊天列表时，如何处理长文本渲染导致的页面卡顿？

## 回答

### 一、问题分析

AI 聊天场景下，长文本渲染卡顿的主要原因包括：

1. **大量 DOM 节点**：长对话历史积累了成百上千的消息气泡
2. **复杂内容渲染**：Markdown、代码高亮、数学公式、图片等混合内容
3. **频繁更新**：流式输出时每个 token 都可能触发重新渲染
4. **复杂样式计算**：CSS 选择器和布局计算开销大

### 二、虚拟滚动（Virtual Scrolling）

虚拟滚动是处理长列表的核心技术，只渲染可视区域内的元素。

#### 基本原理

```javascript
/**
 * 虚拟滚动的核心思想：
 * 1. 计算可视区域能容纳多少条消息
 * 2. 根据滚动位置计算起始索引
 * 3. 只渲染可视区域 + 缓冲区的消息
 * 4. 用 padding 或 transform 模拟完整列表高度
 */
```

#### 使用 react-virtuoso 实现

```jsx
import { Virtuoso } from 'react-virtuoso'

function ChatList({ messages }) {
  return (
    <Virtuoso
      data={messages}
      // 初始滚动到底部
      initialTopMostItemIndex={messages.length - 1}
      // 跟随新消息自动滚动
      followOutput="smooth"
      // 渲染单条消息
      itemContent={(index, message) => (
        <MessageBubble key={message.id} message={message} />
      )}
      // 预渲染缓冲区大小（像素）
      overscan={200}
      // 动态高度支持
      estimateSize={() => 100}
    />
  )
}
```

#### 处理动态高度

聊天消息高度不固定，需要特殊处理：

```jsx
import { Virtuoso } from 'react-virtuoso'
import { useRef, useCallback } from 'react'

function DynamicHeightChatList({ messages }) {
  const virtuosoRef = useRef(null)
  // 缓存已测量的高度
  const heightCache = useRef(new Map())

  // 当消息内容变化时（如图片加载完成），通知虚拟列表重新计算
  const handleContentResize = useCallback((messageId, newHeight) => {
    heightCache.current.set(messageId, newHeight)
    // 触发 Virtuoso 重新计算布局
    virtuosoRef.current?.measure()
  }, [])

  return (
    <Virtuoso
      ref={virtuosoRef}
      data={messages}
      itemContent={(index, message) => (
        <ResizeObserverWrapper messageId={message.id} onResize={handleContentResize}>
          <MessageBubble message={message} />
        </ResizeObserverWrapper>
      )}
    />
  )
}

// 使用 ResizeObserver 监听内容尺寸变化
function ResizeObserverWrapper({ messageId, onResize, children }) {
  const ref = useRef(null)

  useEffect(() => {
    if (!ref.current) return

    const observer = new ResizeObserver((entries) => {
      const height = entries[0]?.contentRect.height
      if (height) {
        onResize(messageId, height)
      }
    })

    observer.observe(ref.current)
    return () => observer.disconnect()
  }, [messageId, onResize])

  return <div ref={ref}>{children}</div>
}
```

### 三、异步分片渲染

对于复杂内容（Markdown、代码高亮），使用分片渲染避免阻塞主线程。

#### 使用 requestIdleCallback 分片渲染

```javascript
/**
 * 将长文本分片渲染
 * @param {string} content - 完整内容
 * @param {function} onChunk - 每个分片渲染完成的回调
 * @param {number} chunkSize - 每片大小（字符数）
 */
function renderInChunks(content, onChunk, chunkSize = 500) {
  const chunks = []
  for (let i = 0; i < content.length; i += chunkSize) {
    chunks.push(content.slice(i, i + chunkSize))
  }

  let currentIndex = 0

  function processNextChunk(deadline) {
    // 在浏览器空闲时处理
    while (currentIndex < chunks.length && deadline.timeRemaining() > 5) {
      onChunk(chunks[currentIndex], currentIndex)
      currentIndex++
    }

    if (currentIndex < chunks.length) {
      // 还有剩余，继续调度
      requestIdleCallback(processNextChunk)
    }
  }

  requestIdleCallback(processNextChunk)
}
```

#### React 中使用 useTransition 降低优先级

```jsx
import { useState, useTransition, useMemo } from 'react'
import { marked } from 'marked'

function MarkdownRenderer({ content }) {
  const [isPending, startTransition] = useTransition()
  const [renderedHtml, setRenderedHtml] = useState('')

  useEffect(() => {
    // 将 Markdown 解析放在低优先级更新中
    startTransition(() => {
      const html = marked.parse(content)
      setRenderedHtml(html)
    })
  }, [content])

  if (isPending) {
    return <div className="skeleton">正在渲染...</div>
  }

  return (
    <div className="markdown-body" dangerouslySetInnerHTML={{ __html: renderedHtml }} />
  )
}
```

#### 流式输出时的增量渲染

```jsx
import { useState, useRef, useCallback } from 'react'
import { marked } from 'marked'

function StreamingMarkdown({ tokens }) {
  const [html, setHtml] = useState('')
  const rafRef = useRef(null)
  const pendingTokensRef = useRef('')

  // 批量处理 token，减少渲染次数
  const processTokens = useCallback(() => {
    if (pendingTokensRef.current) {
      const newHtml = marked.parse(pendingTokensRef.current)
      setHtml(newHtml)
    }
    rafRef.current = null
  }, [])

  // 收到新 token 时
  const appendToken = useCallback(
    (token) => {
      pendingTokensRef.current += token

      // 使用 RAF 合并多次更新
      if (!rafRef.current) {
        rafRef.current = requestAnimationFrame(processTokens)
      }
    },
    [processTokens],
  )

  return <div className="markdown-body" dangerouslySetInnerHTML={{ __html: html }} />
}
```

### 四、Markdown 渲染优化

#### 使用 Web Worker 处理复杂解析

```jsx
// 使用 Worker 的组件
function OptimizedMarkdown({ content }) {
  const [html, setHtml] = useState('')
  const workerRef = useRef(null)

  useEffect(() => {
    // 创建 Worker
    workerRef.current = new Worker(new URL('./markdown.worker.js', import.meta.url))

    workerRef.current.onmessage = (event) => {
      setHtml(event.data.html)
    }

    return () => workerRef.current?.terminate()
  }, [])

  useEffect(() => {
    workerRef.current?.postMessage({
      id: Date.now(),
      content,
    })
  }, [content])

  return <div dangerouslySetInnerHTML={{ __html: html }} />
}
```

### 七、处理异步高度变化

图片、公式等内容加载后会改变高度，需要特殊处理：

```jsx
function SmartScrollContainer({ children }) {
  const containerRef = useRef(null)
  const [shouldAutoScroll, setShouldAutoScroll] = useState(true)
  const lastScrollHeightRef = useRef(0)

  // 监听内容变化，保持滚动位置
  useEffect(() => {
    const container = containerRef.current
    if (!container) return

    const observer = new MutationObserver(() => {
      if (shouldAutoScroll) {
        // 如果用户在底部，继续滚动到底部
        container.scrollTop = container.scrollHeight
      } else {
        // 如果用户在查看历史，保持相对位置
        const heightDiff = container.scrollHeight - lastScrollHeightRef.current
        if (heightDiff > 0) {
          // 内容高度增加，调整滚动位置
          container.scrollTop += heightDiff
        }
      }
      lastScrollHeightRef.current = container.scrollHeight
    })

    observer.observe(container, {
      childList: true,
      subtree: true,
      attributes: true,
    })

    return () => observer.disconnect()
  }, [shouldAutoScroll])

  // 检测用户是否在底部
  const handleScroll = useCallback(() => {
    const container = containerRef.current
    if (!container) return

    const isAtBottom =
      container.scrollHeight - container.scrollTop - container.clientHeight < 50
    setShouldAutoScroll(isAtBottom)
  }, [])

  return (
    <div ref={containerRef} onScroll={handleScroll} className="chat-container">
      {children}
    </div>
  )
}
```

### 八、节流防抖优化

```javascript
import { useCallback, useRef } from 'react'

// 使用 RAF 节流滚动事件
function useRAFThrottle(callback) {
  const rafRef = useRef(null)
  const lastArgsRef = useRef(null)

  return useCallback(
    (...args) => {
      lastArgsRef.current = args

      if (rafRef.current) return

      rafRef.current = requestAnimationFrame(() => {
        callback(...lastArgsRef.current)
        rafRef.current = null
      })
    },
    [callback],
  )
}

// 使用示例
function ChatList() {
  const handleScroll = useRAFThrottle((event) => {
    // 滚动处理逻辑
    calculateVisibleItems(event.target.scrollTop)
  })

  return <div onScroll={handleScroll}>{/* 消息列表 */}</div>
}
```

```javascript
const observer = new PerformanceObserver((list) => {

    frameCount = 0
```

### 十、总结

| 优化策略               | 适用场景          | 效果               |
| ---------------------- | ----------------- | ------------------ |
| **虚拟滚动**           | 长消息列表        | 大幅减少 DOM 节点  |
| **分片渲染**           | 复杂内容解析      | 避免阻塞主线程     |
| **Web Worker**         | Markdown/代码高亮 | 计算移至后台线程   |
| **content-visibility** | 长列表            | 浏览器自动跳过渲染 |
| **懒加载**             | 图片/代码块       | 按需加载重资源     |
| **节流防抖**           | 滚动/输入事件     | 减少事件处理频率   |

核心原则：**减少 DOM 数量、避免主线程阻塞、按需渲染**。根据实际场景组合使用这些优化策略，可以确保 AI 聊天应用在处理长对话时保持流畅。
