## 问题

如果需要客户端在 SSE 流式输出过程中实时中断请求，你会怎么做？

## 回答

### 一、问题背景

在 AI 对话应用中，用户可能在 AI 输出过程中想要中断响应，常见场景包括：

- AI 的回答方向不对，用户想提前停止
- 用户误操作发送了问题，想取消
- 网络较慢，用户不想继续等待
- 用户想重新提问或修改问题

实现优雅的中断机制对用户体验至关重要。

### 二、使用 AbortController 中断 Fetch 请求

现代浏览器提供了 `AbortController` API，可以中断正在进行的 fetch 请求：

```javascript
class ChatService {
  constructor() {
    // 保存当前请求的 AbortController
    this.abortController = null
  }

  // 发起流式请求
  async sendMessage(message, onToken) {
    // 如果有正在进行的请求，先中断
    this.abort()

    // 创建新的 AbortController
    this.abortController = new AbortController()

    try {
      const response = await fetch('/api/chat/stream', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message }),
        // 关键：传入 signal 以支持中断
        signal: this.abortController.signal,
      })

      const reader = response.body.getReader()
      const decoder = new TextDecoder()

      while (true) {
        // 每次读取时检查是否已中断
        const { done, value } = await reader.read()
        if (done) break

        const chunk = decoder.decode(value)
        const lines = chunk.split('\n')

        for (const line of lines) {
          if (line.startsWith('data: ')) {
            const data = line.slice(6)
            if (data === '[DONE]') return

            const parsed = JSON.parse(data)
            onToken(parsed.token)
          }
        }
      }
    } catch (error) {
      if (error.name === 'AbortError') {
        // 用户主动中断，这是预期行为
        console.log('请求已被用户中断')
        return { aborted: true }
      }
      // 其他错误需要抛出
      throw error
    } finally {
      this.abortController = null
    }
  }

  // 中断当前请求
  abort() {
    if (this.abortController) {
      this.abortController.abort()
      this.abortController = null
    }
  }
}
```

### 三、使用 EventSource 时的中断处理

如果使用原生 `EventSource` API，中断方式略有不同：

```javascript
class SSEChatService {
  constructor() {
    this.eventSource = null
  }

  connect(sessionId, onToken, onComplete) {
    // 关闭之前的连接
    this.close()

    this.eventSource = new EventSource(`/api/chat/stream?session=${sessionId}`)

    this.eventSource.onmessage = (event) => {
      if (event.data === '[DONE]') {
        onComplete()
        this.close()
        return
      }

      const data = JSON.parse(event.data)
      onToken(data.token)
    }

    this.eventSource.onerror = (error) => {
      console.error('SSE 错误:', error)
      this.close()
    }
  }

  // 关闭 SSE 连接
  close() {
    if (this.eventSource) {
      this.eventSource.close()
      this.eventSource = null
    }
  }
}
```

### 四、服务端如何感知客户端中断？

仅在客户端中断请求还不够，服务端也需要感知中断以便：

- 停止 AI 模型的推理，节省计算资源
- 释放相关内存和连接

#### Node.js 服务端实现

```javascript
app.post('/api/chat/stream', async (req, res) => {
  // 设置 SSE 响应头
  res.setHeader('Content-Type', 'text/event-stream')
  res.setHeader('Cache-Control', 'no-cache')
  res.setHeader('Connection', 'keep-alive')

  // 标记连接是否已关闭
  let isClosed = false

  // 监听客户端断开连接
  req.on('close', () => {
    isClosed = true
    console.log('客户端断开连接，停止生成')
    // 这里可以调用 AI 服务的取消接口
    cancelAIGeneration(req.sessionId)
  })

  // 模拟 AI 流式生成
  const aiStream = await callAIService(req.body.message)

  for await (const token of aiStream) {
    // 每次写入前检查连接状态
    if (isClosed) {
      console.log('连接已关闭，停止发送')
      break
    }

    res.write(`data: ${JSON.stringify({ token })}\n\n`)
  }

  if (!isClosed) {
    res.write('data: [DONE]\n\n')
    res.end()
  }
})
```

#### 使用 OpenAI SDK 的中断示例

```javascript
import OpenAI from 'openai'

app.post('/api/chat/stream', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream')
  res.setHeader('Cache-Control', 'no-cache')

  const openai = new OpenAI()

  // 创建 AbortController 用于取消 OpenAI 请求
  const controller = new AbortController()

  req.on('close', () => {
    // 客户端断开时，中断对 OpenAI 的请求
    controller.abort()
  })

  try {
    const stream = await openai.chat.completions.create(
      {
        model: 'gpt-4',
        messages: [{ role: 'user', content: req.body.message }],
        stream: true,
      },
      {
        signal: controller.signal, // 传入 signal
      },
    )

    for await (const chunk of stream) {
      const content = chunk.choices[0]?.delta?.content || ''
      if (content) {
        res.write(`data: ${JSON.stringify({ token: content })}\n\n`)
      }
    }

    res.write('data: [DONE]\n\n')
    res.end()
  } catch (error) {
    if (error.name === 'AbortError') {
      // 请求被中断，正常结束
      return
    }
    throw error
  }
})
```

### 五、React 组件中的完整实现

```jsx
import { useState, useRef, useCallback } from 'react'

function ChatComponent() {
  const [messages, setMessages] = useState([])
  const [isGenerating, setIsGenerating] = useState(false)
  const [currentResponse, setCurrentResponse] = useState('')
  const abortControllerRef = useRef(null)

  // 发送消息
  const sendMessage = useCallback(
    async (userMessage) => {
      // 添加用户消息
      setMessages((prev) => [...prev, { role: 'user', content: userMessage }])
      setIsGenerating(true)
      setCurrentResponse('')

      // 创建新的 AbortController
      abortControllerRef.current = new AbortController()

      try {
        const response = await fetch('/api/chat/stream', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({ message: userMessage }),
          signal: abortControllerRef.current.signal,
        })

        const reader = response.body.getReader()
        const decoder = new TextDecoder()
        let fullResponse = ''

        while (true) {
          const { done, value } = await reader.read()
          if (done) break

          const chunk = decoder.decode(value)
          const lines = chunk.split('\n')

          for (const line of lines) {
            if (line.startsWith('data: ') && line.slice(6) !== '[DONE]') {
              const { token } = JSON.parse(line.slice(6))
              fullResponse += token
              setCurrentResponse(fullResponse)
            }
          }
        }

        // 生成完成，添加到消息列表
        setMessages((prev) => [...prev, { role: 'assistant', content: fullResponse }])
      } catch (error) {
        if (error.name === 'AbortError') {
          // 将已生成的部分内容保留
          if (currentResponse) {
            setMessages((prev) => [
              ...prev,
              {
                role: 'assistant',
                content: currentResponse + ' [已中断]',
              },
            ])
          }
        } else {
          console.error('发送失败:', error)
        }
      } finally {
        setIsGenerating(false)
        setCurrentResponse('')
        abortControllerRef.current = null
      }
    },
    [currentResponse],
  )

  // 中断生成
  const stopGeneration = useCallback(() => {
    if (abortControllerRef.current) {
      abortControllerRef.current.abort()
    }
  }, [])

  return (
    <div className="chat-container">
      {/* 消息列表 */}
      <div className="messages">
        {messages.map((msg, idx) => (
          <div key={idx} className={`message ${msg.role}`}>
            {msg.content}
          </div>
        ))}
        {/* 正在生成的消息 */}
        {currentResponse && (
          <div className="message assistant generating">
            {currentResponse}
            <span className="cursor">|</span>
          </div>
        )}
      </div>

      {/* 控制按钮 */}
      <div className="controls">
        {isGenerating ? (
          <button onClick={stopGeneration} className="stop-btn">
            ⏹ 停止生成
          </button>
        ) : (
          <button onClick={() => sendMessage(inputValue)}>发送</button>
        )}
      </div>
    </div>
  )
}
```

### 六、中断请求的最佳实践

#### 1. 及时清理资源

```javascript
// 组件卸载时确保中断请求
useEffect(() => {
  return () => {
    if (abortControllerRef.current) {
      abortControllerRef.current.abort()
    }
  }
}, [])
```

#### 2. 提供明确的用户反馈

```javascript
const stopGeneration = () => {
  if (abortControllerRef.current) {
    abortControllerRef.current.abort()
    // 立即更新 UI 状态，不等待请求结束
    setIsGenerating(false)
    showToast('已停止生成')
  }
}
```

#### 3. 处理快速连续请求

```javascript
// 防止重复点击导致多个请求
const sendMessage = async (message) => {
  // 先中断之前的请求
  if (abortControllerRef.current) {
    abortControllerRef.current.abort()
  }

  // 等待一小段时间确保之前的请求已清理
  await new Promise((resolve) => setTimeout(resolve, 50))

  // 发起新请求...
}
```

#### 4. 服务端资源释放

```javascript
// 使用 WeakMap 跟踪活跃的生成任务
const activeGenerations = new Map()

app.post('/api/chat/stream', async (req, res) => {
  const sessionId = req.headers['x-session-id']

  // 取消该 session 之前的任务
  if (activeGenerations.has(sessionId)) {
    activeGenerations.get(sessionId).abort()
  }

  const controller = new AbortController()
  activeGenerations.set(sessionId, controller)

  req.on('close', () => {
    activeGenerations.delete(sessionId)
    controller.abort()
  })

  // ... 处理请求
})
```

### 七、总结

实现流式输出的中断需要前后端配合：

| 层面         | 关键点                                                               |
| ------------ | -------------------------------------------------------------------- |
| **客户端**   | 使用 `AbortController` 中断 fetch，或 `EventSource.close()` 关闭 SSE |
| **服务端**   | 监听 `req.on('close')` 事件，及时停止 AI 推理并释放资源              |
| **AI 服务**  | 调用 AI SDK 时传入 `signal`，支持中断长时间运行的推理任务            |
| **用户体验** | 提供明确的停止按钮，中断后保留已生成的内容                           |

合理的中断机制不仅提升用户体验，还能节省服务器资源和 API 调用成本。
